# ADR-0007: 사용자 자가 텔레그램 봇 토큰

**Date**: 2026-05-07
**Status**: accepted
**Deciders**: 사용자(태민)

## Context

자가설치 모델(ADR-0006)에서 텔레그램 봇 토큰을 어떻게 관리할지 결정해야 한다. 옵션은 (1) 각자 BotFather에서 발급해 본인 `.env`에 보관, (2) 공유 봇 1개를 배포자가 운영하고 사용자는 chat_id만 등록, (3) 서버가 모든 사용자 토큰을 중앙 보관 — 세 가지다. 시크릿 책임 경계와 운영 비용이 핵심 평가 기준이다.

## Decision

**사용자 각자 BotFather에서 자기 봇을 발급받아 본인 환경에 보관**한다. 배포자(코드 작성자)는 어떤 사용자의 토큰도 보유하지 않는다. README와 GUI 설정 페이지에 BotFather 사용 가이드(스크린샷 포함)를 제공하고, `signal doctor`가 토큰 유효성을 자체 검증한다.

## Alternatives Considered

### Alternative 1: 공유 봇 1개 + 사용자 chat_id 등록
- **Pros**: 사용자는 `/start`로 chat_id만 받으면 됨, 진입 단계 단축
- **Cons**: 배포자가 봇 운영 책임, 봇 정지/장애 시 모든 사용자 영향, 메시지 송신 한도(분당 30/봇)에 묶임
- **Why not**: 자가설치 모델과 결이 안 맞음. 배포자에게 단일 장애점 부여

### Alternative 2: 중앙 토큰 관리 (서버 DB)
- **Pros**: 사용자별 봇 등록을 SaaS로 자동화 가능
- **Cons**: DB 암호화·접근 통제·GDPR/KISA 등 부담, 토큰 유출 시 N명 영향
- **Why not**: v3 SaaS 영역. v1·v2에서 부적절

## Consequences

### Positive
- 시크릿 유출 시 책임이 사용자 본인에게 한정
- 배포자는 사용자 메시지/토큰에 접근 불가 → 개인정보 이슈 회피
- 봇 이름·아이콘·설명을 사용자가 자유롭게 설정 가능
- 텔레그램 봇 한도(메시지 송신율)가 사용자별로 독립 → 다중 사용자에서 폭주 없음

### Negative
- BotFather 사용 단계가 신규 사용자에게 진입 비용(5분 정도)
- README에 스크린샷 가이드 필수
- 사용자가 토큰을 분실/유출 시 본인이 재발급해야 함

### Risks
- 비개발자 또는 텔레그램 미숙 사용자에게 BotFather가 장벽이 될 수 있음
- **완화**: README에 단계별 스크린샷, GUI 설정 페이지에 "봇 만들기 가이드 보기" 링크
- **재검토 시점**: SaaS(v3) 전환 시 본 ADR을 superseded 처리
