# ADR-0001: BB + CCI 지표 조합 채택

**Date**: 2026-05-07
**Status**: accepted
**Deciders**: 사용자(태민)

## Context

업비트 KRW 메이저 코인의 1시간봉 시그널 시스템에서 어느 차원의 지표를 결합할지 결정해야 한다. 변동성·모멘텀·추세·통계적 이탈 등 후보가 많고, 1시간봉 KRW 메이저는 박스권과 추세장이 혼재하는 특성이 있어 단일 지표만으로는 false signal이 많다. 두 개 이상의 지표가 동시에 합의해야 신뢰도가 올라간다.

## Decision

**볼린저 밴드(BB-20-2)** 와 **CCI(20)** 두 지표를 결합해 사용한다. 두 지표가 동시에 같은 방향을 가리킬 때만 시그널을 발생시키고, 동일한 두 지표 위에서 평균회귀(모드 A)와 추세 돌파(모드 B) 두 모드를 분기한다.

## Alternatives Considered

### Alternative 1: RSI + MACD
- **Pros**: 추세장 강함, 학습 자료 풍부, 모멘텀+추세 차원 합의
- **Cons**: 변동성 차원 부재, 과매수/과매도 동시 검출 부족
- **Why not**: 1시간봉 KRW 메이저의 박스권 구간에서 BB가 빠지면 진입점 정의가 약해짐

### Alternative 2: BB + RSI
- **Pros**: 직관적, 변동성+모멘텀 합의
- **Cons**: RSI 정규화로 인해 극단값 빈도가 낮음 → 강한 시그널 적음
- **Why not**: CCI 대비 변별력이 떨어짐

### Alternative 3: ATR + ADX + BB
- **Pros**: 추세 필터 강력, 변동성+추세 강도 동시 측정
- **Cons**: 지표 3개 → 시그널 빈도 너무 낮음, 코드 복잡도↑
- **Why not**: v1 단순성 우선 (KISS)

## Consequences

### Positive
- 단일 지표 모듈에서 두 모드(평균회귀·돌파)를 동시 평가 → 코드 중복 없음
- 두 지표 합의 조건으로 false signal 감소
- BB·CCI 모두 계산이 단순하고 외부 레퍼런스(TradingView)와 비교 가능

### Negative
- BB·CCI 모두 지연 지표 → 진입 타이밍이 늦을 수 있음 (24봉 보유 또는 BB 중심선 도달 청산 규칙으로 일부 완충)
- 강한 추세장에서 모드 A는 false signal 발생 가능 → 모드 B 거래량 1.5배 필터로 보완

### Risks
- 박스권/추세 자동 판별이 없어 시장 상태 변화 시 적중률 저하 가능
- **재검토 시점**: 1개월 라이브 데이터 누적 후 모드별 적중률 비교. 필요 시 4시간봉 추세 필터 추가(ADR-0004 재검토 연계).
