# ADR-0002: 자동매매 제외

**Date**: 2026-05-07
**Status**: accepted
**Deciders**: 사용자(태민)

## Context

시그널을 자동으로 주문 실행까지 연결할지 결정해야 한다. 업비트는 인증 API로 주문이 가능하지만, 잘못된 시그널이 즉시 자금 손실로 이어지고 슬리피지/체결 실패 처리·API 키 노출 등 책임 경계가 모호한 위험이 추가된다. v1의 핵심 가치는 **신호 품질 검증**이지 자금 운용이 아니다.

## Decision

v1·v2에서 **자동매매를 하지 않는다**. 텔레그램 알림 + 사용자 수동 매매로 한정한다. 신호 품질이 검증된 후 별도 PRD로 v3에서 1-click 텔레그램 주문부터 단계적 도입을 검토한다.

## Alternatives Considered

### Alternative 1: 알림 + 자동매매
- **Pros**: 사용자 개입 불필요, 24/7 자동 운용
- **Cons**: 잘못된 시그널이 즉시 자금 손실로 직결, API 키 노출 보안 위험, 슬리피지/체결 실패 핸들링 복잡, 책임 경계 모호
- **Why not**: 신호 품질이 검증되지 않은 v1에서 자금 위험을 더하면 안 됨

### Alternative 2: 알림 + 1-click 텔레그램 주문(인라인 키보드)
- **Pros**: 사용자 확인 단계가 있어 자동매매보다 안전, 모바일 즉시 실행
- **Cons**: API 키 보관 필요, 보안 표면 증가, 사용자 한 번 잘못 누르면 체결
- **Why not**: 신호 적중률이 검증된 뒤 v3에서 별도 PRD로 다룰 영역. v1·v2 스코프 외.

## Consequences

### Positive
- 업비트 인증 API 키 불필요 → 보안 표면 최소(텔레그램 토큰만)
- 잘못된 시그널이 자금 손실로 이어지지 않음
- 책임 경계 명확: 매매 결정은 사용자 본인

### Negative
- 사용자가 매번 텔레그램 → 업비트 앱 전환해 직접 주문해야 함
- 빠른 진입이 필요한 시그널을 놓칠 수 있음

### Risks
- 자동매매를 원하는 사용자 요구가 누적되면 v3 SaaS PRD로 이관해 보안 리뷰와 함께 다룰 것
- **재검토 시점**: 라이브 시그널 적중률 ≥ 60% 누적 시 v3 1-click 주문 PRD 작성
