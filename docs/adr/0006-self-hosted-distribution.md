# ADR-0006: 오픈소스 자가설치 배포

**Date**: 2026-05-07
**Status**: accepted
**Deciders**: 사용자(태민)

## Context

"다른 사용자가 사용할 수 있게"의 해석은 자가설치 / LAN 공유 / SaaS 세 갈래로 나뉘며, 각각 보안·운영 부담이 다르다. SaaS는 인증·DB·결제·이용약관·개인정보 처리·중앙 봇 관리·법적 책임까지 별도 v3 프로젝트 규모가 된다. 자가설치는 사용자 환경에서 격리되어 시크릿 책임이 분리된다.

## Decision

**GitHub 공개 + 사용자 자가설치** 모델로 시작한다. 각 사용자가 자기 PC에 설치해 자기 텔레그램 봇 토큰으로 운영한다. 배포자(코드 작성자)는 어떤 사용자의 토큰·시그널 데이터에도 접근하지 않는다. 클라우드 호스팅·회원가입은 별도 v3 프로젝트로 분리.

## Alternatives Considered

### Alternative 1: LAN 소규모 공유 서버
- **Pros**: 1대 서버(라즈파이 등)에서 가족·지인 공유
- **Cons**: 서버 운영자 1명에게 시크릿 책임 집중, LAN 인증 추가 필요
- **Why not**: 자가설치 대비 명확한 이점 없음. 사용자 풀 5명 이내라는 가정도 자가설치로 충분

### Alternative 2: SaaS (인터넷 호스팅, 회원가입)
- **Pros**: 사용자 진입 장벽 가장 낮음, 모르는 사람도 가입 가능
- **Cons**: 인증·DB·결제·이용약관·개인정보(KISA/GDPR)·중앙 봇 관리 모두 필요. 운영 비용↑
- **Why not**: v1 스코프 초과. 신호 품질 검증 후 별도 PRD로 다룰 영역

### Alternative 3: 비공개 + 본인만 사용 (v1.0 모델 유지)
- **Pros**: 가장 단순
- **Cons**: 사용자 확장 0
- **Why not**: v2.0 목표 미달

## Consequences

### Positive
- 인증·DB·결제 모두 불필요 → 코드와 운영 단순
- 시크릿(텔레그램 토큰)이 사용자 로컬 `.env`에만 존재 → 배포자 책임 면제
- 사용자별 데이터 격리 자동 보장
- 라이선스(MIT 권장) 적용 시 다른 개발자 기여 가능

### Negative
- 신규 사용자가 직접 Python·uv·텔레그램 봇 발급을 해야 함
- README/Quick Start 가이드의 품질이 사용자 경험을 결정 (G6: 30분 이내 첫 시그널 목표)
- 비개발자 진입은 어려움 (개발 경험 1년 이상 가정)

### Risks
- 사용자가 잘못 설정해 외부 노출(`WEB_BIND=0.0.0.0` + 빈 비번)하면 시크릿 노출 위험 → 시작 거부 가드 필수(ADR-0005)
- **재검토 시점**: GitHub Issue로 "호스팅 서비스 원함" 요청 누적 시 v3 SaaS PRD 검토
