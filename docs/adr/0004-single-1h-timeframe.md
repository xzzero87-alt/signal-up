# ADR-0004: 1시간봉 단일 타임프레임

**Date**: 2026-05-07
**Status**: accepted
**Deciders**: 사용자(태민)

## Context

시그널 평가 기준 봉을 결정해야 한다. 너무 짧으면 노이즈로 false signal이 폭증하고 알림 피로도가 높아진다. 너무 길면 시그널 빈도가 너무 낮아 도구 가치가 떨어진다. 멀티타임프레임(추세 필터)은 정확도를 올리지만 봉 정합/시점 동기 로직이 추가되어 v1에서 불필요한 복잡성이 된다.

## Decision

**1시간봉 단일 타임프레임**으로 시작한다. 멀티타임프레임은 v1에서 도입하지 않고, 1개월 라이브 데이터 누적 후 false signal 패턴을 보고 4시간봉 추세 필터 추가를 검토한다.

## Alternatives Considered

### Alternative 1: 15분봉 단일
- **Pros**: 민감도 높음, 빠른 진입
- **Cons**: 노이즈↑, 알림 빈도 너무 잦아 피로도↑, false signal 비율↑
- **Why not**: 데이트레이딩이 목적이 아니고, 알림 도구로서 빈도가 부적절

### Alternative 2: 일봉 단일
- **Pros**: 노이즈 매우 낮음, 백테스트 빠름
- **Cons**: 시그널 빈도 너무 낮음(코인당 월 1~2회), 도구 가치 떨어짐
- **Why not**: 스윙 트레이딩에 적합하나 시간 단위 의사결정에 부적합

### Alternative 3: 멀티타임프레임 (4h 추세 + 1h 진입)
- **Pros**: 정확도↑, 추세 방향 합의
- **Cons**: 봉 정합/시점 동기 로직 필요, 백테스트 속도 저하, 코드 복잡도↑
- **Why not**: v1 KISS 원칙. 단일 봉으로 운영하며 false signal 패턴이 보이면 v2에서 추세 필터로 추가

## Consequences

### Positive
- 단일 시계열만 다루어 코드·백테스트 단순
- 시그널 빈도 적당(코인당 주 1~5회)
- 1시간봉은 노이즈와 빈도의 균형이 좋음

### Negative
- 횡보장 추세 반전에서 false signal 가능 → 모드 B 거래량 1.5x 필터로 일부 보완
- 강한 추세장에서 모드 A는 역추세 진입 위험

### Risks
- 1개월 운영 후 false signal 비율이 높으면 추세 필터 추가 필요
- **재검토 시점**: 1개월차 회고에서 4시간봉 MA 또는 ADX 추세 필터 추가 ROI 평가
